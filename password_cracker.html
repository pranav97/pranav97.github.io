Multithreaded Distributed Password Cracker

Problems can have many solutions. In the case of the Number of Islands problem there are at least 3 different solutions.

Problem Statement

Inputs: N number of lowercase 6 character encoded passwords that were generated using crypt in standard library.

Example:
Input:
an47aiHfl8mYU
dfnPloK4B2dE2
mmgThmekP3YfA
ilWX9JLsMJN/M
bri2PbcuqAXeA

Output: There was a competition among the people in the class for who can crack the passwords fastest.

Example output:
pranav
asdfgh
hammer
brilli
anbran

The goal was to brute force the password (that is to try every single password combination until we get the password hash that was provided starting at aaaaaa to zzzzzz. The fastest password crack would use all the 4 CPUs and all the 24 cores each (96 total cores) solving the problem as efficiently as possible.

Truely Parallel

IF the number of passwords to be cracked is greater than the number of cores. The task of cracking these n passwords is perfect for a truly parallel algorithm in which core would get n / 96 passwords. For example if the number of passwords was 96 * 3 (288) then each core can do 3 passwords. For tasks in which the subtasks are similarly sized, a truly parallel solution might be sufficient.

Embarrassingly parallel

The issue was that some of the passwords were "easier" than others. So we came up with ways to split the search space of aaaaaa to zzzzzz among the different codes and had the thread with the password to terminate all the threads that are still searching for the password.

Union Find or Disjoint-set

Union Find is a data structure that tracks a set of elements partitioned into disjoint or non-overlapping subsets. This technique of dividing the components into non-overlapping sets can be used in many problems. For example,

  • Finding out the number of connected components in a digital picture.
  • Finding out if there exists a path between two nodes in a network.
  • Kruskal's algorithm - This greedy algorithm finds the minimum weight spanning tree for a connected graph. A minimum weight spanning tree has all the vertices of the fully connected graph. More importantly, the algorithm minimizes the weights in the graph. In a network situation, the edges in the minimum weight spannning tree might be used over edges that are not since those links have the least cost (take the least time).

  • For more information about the union find algorithm, click here.

In the case of this grid, all of the land masses or 1s are going to be elements in the disjoint-set data structure. After joining or unifying neighboring sets in the grid, the number of islands in the grid can be found out.
